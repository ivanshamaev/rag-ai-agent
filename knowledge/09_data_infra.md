# Инфраструктура данных для RAG и ML

Обзор требований к инфраструктуре данных для поддержки пайплайнов RAG и ML-платформ.

## Хранилище данных

- **Данные сырья**: S3/Blob, файловые хранилища, базы данных.
- **Метаданные**: schema registry, таблицы с наборами данных, версии.

## Инструменты оркестрации

- Apache Airflow, Prefect — для управления ETL/ингестом и расписанием задач.

## Каталогизация и governance

- Data catalogs, lineage: отслеживание происхождения данных, политики доступа.

## Безопасность и приватность

- Маскирование данных, контроль доступа и шифрование при хранении и передаче.

## Инфраструктура для embeddings

- Поддержка масштабного вычисления эмбеддингов (batching, GPU), хранение и реиндексация.

## Пояснение для новичка: зачем нужна инфраструктура данных

- **Почему это важно**: без правильно настроенной инфраструктуры ваши данные будут разбросаны, устаревать или быть недоступными для модели.
- **Ingestion pipeline**: последовательность шагов от получения данных до их сохранения и подготовки для индексирования.
- **Lineage / Tracing**: позволяет понять, откуда пришёл конкретный фрагмент и как он изменялся — важно для аудита и исправления ошибок.

Пример простого сценария: вы загружаете новые документы раз в неделю, запускаете пайплайн, который очищает текст, разбивает на чанки, генерирует эмбеддинги и обновляет векторную базу — пользователи сразу видят новые ответы.
