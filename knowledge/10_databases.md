# Типы баз данных: реляционные и NoSQL

Краткий обзор особенностей SQL и NoSQL баз в контексте ML и RAG.

## Реляционные (SQL)

- **Особенности**: строгая схема, транзакции (ACID), индексы (B-tree, GiST), сильная целостность.
- **Плюсы**: предсказуемость, зрелые инструменты, удобны для OLTP и аналитических витрин (via materialized views).
- **Минусы**: сложнее масштабировать горизонтально.

## NoSQL

- **Типы**: документные (MongoDB), key-value (Redis), wide-column (Cassandra), графовые.
- **Плюсы**: гибкие схемы, горизонтальное масштабирование, высокая пропускная способность для некоторых нагрузок.
- **Минусы**: слабее консистентность, разные модели запросов.

## Индексы и оптимизация

- Выбирайте индекс под нагрузку: B-tree для равенств/диапазонов, inverted index для full-text, специализированные индексы для гео/JSON.

## Рекомендации

- Храните источники истины (master data) в надежной системе с бекапами.
- Для быстрых метаданных и кэширования используйте Redis/ElastiCache.

## Пояснение для новичка: как выбрать базу данных

- **Если данные структурированы и важны транзакции** (например, финансовые операции) — выбирайте SQL.
- **Если данные гибкие, часто меняются и нужна масштабируемость** — NoSQL может быть проще.
- **Для быстрых операций и кэша** используйте key-value базы (Redis).

Пример: под хранение исходных документов и метаданных хорошо подойдёт объектное хранилище (S3), а для быстрых мета-запросов — реляционная БД + Redis для кэширования.
