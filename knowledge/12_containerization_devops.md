# Контейнеризация и DevOps для ML

Ключевые аспекты развёртывания ML и RAG-сервисов с использованием контейнеров, CI/CD и мониторинга.

## Docker

- **Образы и Dockerfile**: принципы написания лёгких и безопасных образов.
- **Сетевые аспекты**: порты, сетевые мосты, зависимые сервисы (DB, vector DB).

## CI/CD

- **Концепция**: автоматизация сборки, тестирования, деплоя.
- **Инструменты**: GitHub Actions, GitLab CI, Jenkins.
- **Практики**: тесты интеграции, канареечные релизы, откаты.

## Мониторинг и observability

- **Метрики**: latency, throughput, error rate, success rate, model-specific metrics (prediction distribution).
- **Инструменты**: Prometheus, Grafana, OpenTelemetry.
- **Логи и трейсинг**: централизованные логи (ELK stack), распределённый трейсинг (Jaeger).

## ML-специфика

- Мониторинг деградации модели (data drift, concept drift).
- Автоматизация ре-тренинга и CI для моделей (Model CI/CD).

## Пояснение для новичка: базовые понятия и первые шаги

- **Контейнер (Docker)**: упакованный образ приложения со всеми зависимостями. Контейнеры помогают запускать приложение одинаково везде.
- **CI (Continuous Integration)**: автоматическая сборка и тестирование каждого изменения в коде.
- **CD (Continuous Deployment/Delivery)**: автоматический деплой изменений в окружение (часто сначала в staging, затем production).
- **Observability**: не просто логирование, а система мониторинга, трейсинга и метрик, которая показывает, как работает приложение в реале.

Простой старт: создайте `Dockerfile` для вашего сервиса, подключите GitHub Actions для автоматической сборки образа и отправки в registry (например, Docker Hub), и установите базовый мониторинг через Prometheus + Grafana.
